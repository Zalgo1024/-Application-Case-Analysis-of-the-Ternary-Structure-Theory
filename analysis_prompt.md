# 三元结构理论分析生成指南 (v6.0 — 案例驱动版)

## 核心升级原则

v6.0 对 v5.0 的根本改变：**概念不再是从维度菜单里预选的透镜，而是从案例土壤中长出来的分析工具。** 先看案例发生了什么，识别出需要解释的动态模式，再到理论概念池中匹配最能解释这些模式的概念。

v5.0 保留的核心：核心命题贯穿、接驳语衔接、汇流段结论、可传播金句——这些机制仍然有效，只是概念的产生路径变了。

---

## 一、写报告前的必做事项：案例诊断 → 概念自选 → 命题生成

**在写任何章节之前，先完成以下四步（顺序不可跳越）：**

### Step 1：案例事实诊断

不选概念、不定维度，先纯粹地"看案例"。回答三个问题：

1. **核心张力是什么？** 用一句话写出案例中"最说不通"的地方——读这句话时理性直觉反应应是"咦，这不对"
2. **有哪些反复出现的动态模式？** 列出案例中观察到的行动模式、互动模式、结构模式（不限数量，先列出来）。例如：某人持续付出却持续得不到回报；某群体一触即发、一点就着；某叙事被选择性截取后引爆舆论
3. **哪些事实最需要解释？** 标记出"为什么会这样"的事实——这些是概念选择的原点

**输出**：一份案例诊断清单（不进入最终报告，仅作为分析前的思维准备）

### Step 2：概念自选

拿着 Step 1 识别出的"需要解释的模式"，到三元结构理论的**完整概念池**中匹配：

| Step 1 识别的动态模式 | 匹配的概念 | 为什么这个概念能解释这个模式 |
|---|---|---|
| 模式A | 概念X | ... |
| 模式B | 概念Y | ... |
| 模式C（如有） | 概念Z | ... |

**概念池（全量可用，不按维度分组，跨维度自由匹配）：**

| 概念 | 所属维度 | 一句话释义 | 典型适用场景 |
|---|---|---|---|
| 物质/社会/精神三维度 | 生存 | 生存不止温饱，还有身份和意义 | 个体或群体在哪一层"活不下去" |
| 成本-收益核算 | 生存 | 投入大于回报的持续失衡 | 谁在一直亏、亏什么、亏了多久 |
| 代偿 | 生存 | 用A维度的过度付出来弥补B维度的亏空 | 明知不对却停不下来的行为 |
| 异化 | 生存 | 代偿固化后自我与行为脱节 | "我已经不是我了"的状态 |
| 相变 | 生存 | 量变积累到临界点后质变 | 突然崩溃、突然爆发、突然翻转 |
| 复制性/变异性/时间性 | 繁衍 | 传递什么、怎么变、窗口多长 | 传承、扩散、传播、制度延续 |
| 四载体 | 繁衍 | 生物/文化/制度/技术四种传递载体 | 哪种载体在起作用、哪种失效 |
| 三存续边界 | 繁衍 | 传递能撑多久 | 传统为何断裂、制度为何僵化 |
| 三性质（建设/消极/破坏） | 逆反 | 逆反的结果是修还是毁 | 抗议、躺平、暴力对抗的性质判断 |
| 五类型（主动/被动/自发/协从/主导） | 逆反 | 谁在反、怎么反 | 个体觉醒 vs 群体裹挟 |
| 激活阈值 | 逆反 | 压死骆驼的最后一根稻草 | 什么事触发了逆反、为什么是这件事 |
| 逆反失能 | 逆反 | 该反没反、反了没用 | 沉默的大多数、制度化的无力 |
| 火药桶效应 | 逆反 | 积压→引爆→反噬 | 群体性事件、网络暴力、集体宣泄 |
| 六大利益类型 | 利益 | 物质/安全/政治/身份文化/制度性未来/公共利益 | 谁在争什么 |
| 利益转化路径 | 利益 | 政治权力→经济收益→文化影响力…… | A如何变成B |
| 利益动线 | 利益 | 经济/权力/文化/法律四路径追踪 | 资源从哪来、流向谁、谁卡住 |
| 叙事战场 | 利益 | 谁控制了"真相定义权" | 信息操纵、舆论引导、选择性公开 |
| 四大历史动力机制 | 利益 | 激励-约束/嵌入-制度化/合法化-叙事化/连锁-外溢 | 短期如何变长期、局部如何扩散 |
| 三元互构关系 | 利益 | 生存-繁衍-逆反三维度互为因果 | 维度间的联动放大效应 |
| 三点式框架 | 通用 | A持存/B延伸/C偏离与重构 | 抽象分析、不想贴维度标签时 |

**选择规则：**
- 最多选 3 个概念（特殊情况不超过 4 个）
- **概念可以跨维度自由组合**——不要求先选维度再从维度里挑概念
- 每个概念必须能回答"Step 1 中的哪个模式需要它来解释"——如果答不出，不选
- 不选"应该选"的概念，只选"案例需要"的概念
- 用户明确指定维度时，优先在该维度内匹配，但仍以案例模式为最终判据

### Step 3：从案例诊断 + 概念匹配中提炼核心命题

核心命题不是凭空构思的——它从案例事实与理论概念的碰撞中涌现：

1. 案例中最反常识的事实 × 最能解释它的概念 = 核心命题的雏形
2. 用不超过 40 个字凝练为一句话
3. 检验：选出的概念是否从不同角度共同支撑这个命题？

核心命题要求（同 v5.0）：
- 能引发争议或好奇（不是常识，不是废话）
- 包含一个"为什么"或"是什么"的深层判断

示例：
- ✅ 「彩礼不是爱情的对价，而是一套失效规范系统的最后账单」
- ✅ 「斩杀线真正划出的不是贫富，而是能不能从风险中恢复的能力边界」
- ✅ 「胖猫死后被所有人消费了一遍——唯独没有人追问，一个21岁的男孩为什么会走到长江大桥上」
- ❌ 「本案反映了社会转型期的多重矛盾」（废话）
- ❌ 「合法性松动导致社会撕裂」（套话）

### Step 4：概念间接驳设计

同 v5.0：每个概念之间必须有逻辑衔接——上一个概念得出的结论，如何引出下一个概念要回答的问题。

示例：「知道了谁在精神层面持续亏空（概念一），还需要追问：亏损的信号传出去后，是谁第一个拿到了定义"为什么亏"的权力——这就是叙事战场要揭开的问题（概念二）。」

## 二、六部分固定结构（v6.0 改动项用★标注）

### 第一部分：封面页
（引擎自动生成，无需手写）

### 第二部分：★案例事实摘要（前置！）

**v6.0 核心变化：事实摘要从"第四部分"提前到"第二部分"。逻辑：先看案例发生了什么，再决定用什么概念分析。**

- 时间线 + 关键数据 + 争议焦点
- 纯事实，不夹带分析，不夹带理论术语
- 关键事实或原话使用引用块（>）突出
- 每个事实都要是后续分析的"原材料"，不是装饰

### 第三部分：★分析框架说明（案例驱动版）

**v6.0 核心变化：概念说明不再"先列概念再写分析问题"，而是"先说案例中发现了什么模式，再说明为什么选择这个概念来解释"。读者看到的是：事实在先，概念在后。**

格式：

```
> 核心张力：[一句话写出案例中"最说不通"的地方]

**本报告核心命题：[一句话命题]**

基于对案例事实的诊断，本报告识别出以下需要解释的动态模式，并选择相应的理论概念进行分析：

| 案例中观察到的模式 | 选用的概念 | 概念如何解释该模式 | 本节分析问题 |
|---|---|---|---|
| [模式A：用案例中的具体事实描述] | [概念X] | [为什么X能解释A] | [本节要回答的具体问题] |
| [模式B] | [概念Y] | [为什么Y能解释B] | [本节要回答的具体问题] |
| [模式C] | [概念Z] | [为什么Z能解释C] | [本节要回答的具体问题] |
```

如果某个概念对普通读者有理解门槛，在此处附一段**功用说明**（不超过80字），仅说清"这个概念在本分析中扮演什么角色"——概念的具体定义随分析正文中的案例事实自然带出。禁止在框架说明中重复概念定义，禁止另设独立的"概念解读"章节。

### 第四部分：★三元结构分析正文（案例驱动版）

**v6.0 核心变化：每节的写作方向从"概念→案例"变为"案例→概念"。**

**每一节的标准结构：**

```
【子标题】（冲突式/反常识式/问句式，不超过15字）
[接驳语段落 - 1-2句，说明这节和上节的逻辑关联]
[案例事实层 - 先呈现该节要分析的案例事实/模式，2-3行]
[概念解释层 - 引入理论概念，解释这个模式为什么会这样运作，2-3行]
[子结论句 - 以 → 开头，1句话，说明这层证据对核心命题的贡献]
```

**关键区别（v5.0 vs v6.0）**：
- v5.0 写法：先说概念是什么 → 再用案例印证概念
- v6.0 写法：先呈现案例中"怎么回事" → 再引入概念解释"为什么会这样"

示例对比：
- ❌ v5.0：**火药桶效应**是指积压不满被触发后集中爆发。胖猫事件中，网民……
- ✅ v6.0：长江大桥上堆了 94.6 吨外卖——这些外卖的主人从未认识胖猫，却每个人都觉得自己"欠他一个公道"。**火药桶效应**解释了这种"一人之死撬动全国情绪"的机制：……

**节间接驳语的写法规范（强制）：**

同 v5.0：
- 第一节：可以直接进入（无需接驳语）
- 第二节开头：必须有一句接驳语，格式参考：
  - 「[第一节子结论的简要概括]——但这只回答了"X"，还没回答"Y"。这正是[概念二名称]要揭开的问题。」
- 第三节开头：必须有一句接驳语，格式参考：
  - 「[前两节的合并逻辑]。知道了这两点，我们还缺最后一块拼图：[概念三名称]。」

**示意图要求：**
- ◇ 有条件推荐配图：当分析涉及多方主体互动、层级关系、利益流动、动态演变等"可视化能增值"的场景时，使用 DIAGRAM JSON 格式生成关系网络图或利益动线图。若分析内容为纯逻辑推演、文字已能完整表达机制，可跳过配图。配图与否取决于内容需求，不强制每节必配。
- 图表使用 JSON 数据格式生成，引擎自动渲染为 SVG/PNG 图片
- 图表数据格式：
  ```
  ```DIAGRAM
  {"viz": "network", "title": "图表标题",
   "nodes": [
     {"id": "A", "label": "主体名称", "type": "political", "detail": "详细说明"}
   ],
   "edges": [
     {"source": "A", "target": "B", "label": "动线说明", "type": "economic"}
   ]}
  ```
  ```
- 节点类型（type）：material（物质）/ security（安全）/ political（政治）/ identity_culture（身份文化）/ institutional_future（制度性未来）/ public（公共利益）
- 动线类型（type）：economic（经济路径·实线·金色）/ power（权力路径·长虚线·紫色）/ cultural（文化路径·短虚线·橙色）/ legal（法律路径·点划线·蓝色）
- 图表将自动渲染为：交互式 HTML（支持缩放/拖拽/点击查看详情）+ 静态 PNG（嵌入Word/PDF）
- 回退：若无法生成 JSON 数据，仍可使用纯文本字符画（用 →↓←↑↔ 标示流向）

### 第五部分：结论（同 v5.0）

结论部分禁止用"三个概念并列复述"的格式。

**结论必须包含以下三个层次，顺序不能乱：**

**层次一：汇流段（必写）**
- 用 2-4 句话说清：三个概念分别从哪个角度，共同指向了同一个核心命题
- 格式：「[概念一]告诉我们X，[概念二]告诉我们Y，[概念三]告诉我们Z。三者合指同一个事实：[核心命题的完整表述]。」

**层次二：核心判断（必写）**
- 1 个段落，直接给出对案例的最终判断
- 不能是中性描述，必须有明确的"这意味着什么"

**层次三：可传播金句（必写）**
- 最后 1-2 句，这份报告的"唯一记忆点"
- 必须能独立成立（脱离报告上下文后仍然有力量）
- 引用块（> ）格式

**禁止的结论写法：**
- ❌ 「综上所述，三个维度分别反映了……」（并列复述）
- ❌ 「由此可见，本案具有深远的社会意义」（空话）
- ❌ 「对于……，我们应当……」（建议体）

数据来源、参考文献等。必含版权声明：

## 三、叙事风格铁律（持续遵循）

1. **核心命题贯穿**：每节子结论句必须能被"核心命题"统摄——读者应能感受到，每节分析都在加固同一堵墙
2. **事实驱动**：每一句分析必须引用案例中的具体事实，禁止悬空论断
3. **冲突式子标题**：标题必须能单独引发思考，拒绝平铺直叙
4. **节奏留白**：连续 3 个标准段落后，插入一个 1-2 行的短句或问句
5. **反事实推演**：至少有一次"如果……那么……"的有约束假设（不超过 3 句）
6. **概念限额**：总 ≤ 3 个概念（特殊情况 ≤ 4 个），跨维度自由组合
7. **价值中立**：不使用评价性词汇，仅描述机制、动力、阻力、趋势
8. **人称统一**：统一用"本报告"或"本文"，禁止用"我/我们/笔者"
9. **★案例先行**：每节先呈现案例事实，再引入概念解释——禁止"概念开路、案例佐证"的写法
10. **★禁止分层列举**：禁止在分析段落中使用"第一层…第二层…第三层…"或"首先…其次…最后…"式的分层结构来解释概念机制——这是AI标准句式，读起来像教科书而非深度分析。替代写法：从案例事实出发→概念解释→揭示因果的叙事推进

## 四、利益维度快速参考（基于利益分析 Skill v2026-06-09）

当案例诊断中发现利益争夺模式时，按以下路径展开分析：

### 4.1 六大利益类型快速识别

| 类型 | 核心关键词 | 分析重点 |
|------|---------|---------|
| **物质利益** | 资源、财富、产权、贸易、资本 | 谁控制「抓—运—分—护—续」链条？谁卡住瓶颈节点？ |
| **安全利益** | 威胁、防御、边界、安全措施 | 是真实防御还是「以安全为名的利益转移」？ |
| **政治利益** | 权力、规则制定、制度、职位 | 谁掌握资源分配的水渠？制度优势是否已路径依赖？ |
| **身份文化利益** | 认同、民族、宗教、符号、尊严 | 身份认同如何被激活为动员力？符号如何成为资本？ |
| **制度性未来利益** | 产权保护、法律、继承、规则固化 | 当前制度选择将在多远未来显现？后来者有无翻盘空间？ |
| **公共利益** | 公共、共享、非排他、共识 | 谁在定义「公共利益」的边界？叙事战场如何划定「我们」和「他们」？ |

### 4.2 利益动线四路径追踪（可视化渲染）

```
经济路径：资金/资源/商品 从哪来 → 流向谁 → 如何再投资
  [渲染为实线，金色，箭头指向资金流动方向]

权力路径：规则制定权 在谁手里 → 谁重新分配资源
  [渲染为长虚线，紫色，箭头指向权力行使方向]

文化路径：什么叙事被激活 → 谁在塑造认同
  [渲染为短虚线，橙色，箭头指向叙事扩散方向]

法律路径：什么规则提供边界 → 谁设计、为谁服务
  [渲染为点划线，蓝色，箭头指向规则约束方向]
```

利益动线可视化规则：
- 每条动线在图表中自动渲染为带箭头的曲线，颜色与路径类型对应
- 多条同向边自动偏移，避免重叠
- 动线标签显示在曲线中段
- 支持动画效果：虚线沿箭头方向流动

### 4.3 利益转化五大路径

```
政治权力 ──→ 经济收益（通过法律/许可/合约固化）
经济实力 ──→ 文化影响力（通过赞助/传媒/符号权力）
文化认同 ──→ 资源争夺能力（通过动员/凝聚/牺牲）
各类利益 ──→ 公共利益（溢出个体边界，成为共享沉淀）
制度安排 ──→ 长期优势（把一次性博弈成果固化为结构性优势）
```

### 4.4 四大历史动力机制（深度分析叠加）

| 机制 | 核心公式 | 适用场景 |
|------|---------|---------|
| **激励—约束** | 激励=水流；约束=河道，共同决定行动形式和持续性 | 分析行动者为何这样行动 |
| **嵌入—制度化** | 制度化=筑坝，形成路径依赖 | 分析短期博弈如何固化为长期优势 |
| **合法化—叙事化** | 叙事=给权力涂「正义」油彩；合法化=装订成册跨时间生效 | 分析话语合法性如何被建构 |
| **连锁—外溢** | 没有孤立行动，每次争夺都通过多层网络放大 | 分析局部变化如何扩散为系统性影响 |

### 4.5 利益维度触发的核心命题示例

- ✅ 「这场合作的本质不是双赢，而是政治利益借经济外壳完成转化的单向管道」
- ✅ 「制度的名字叫公平，功能叫锁定——它保护的从来是率先入场的那批人」
- ✅ 「公共利益叙事的崩塌，不是因为没人信，而是因为太多人知道谁在从中得利」

## 六、配图政策（v6.1 新增）

### 6.1 两类图片来源，规则不同

| 路径 | 来源 | 触发条件 | 版权处理 |
|------|------|---------|---------|
| **编码生图** | DIAGRAM JSON → SVG/PNG（引擎自绘） | 按需生成，不受限制 | 无版权问题，作品属分析框架衍生 |
| **网络扒图** | Wikimedia Commons / 官媒配图 / Unsplash | 仅在用户明确要求时执行 | 链式降级验证 → 标注来源 → 无法验证则跳过 |

### 6.2 网络扒图规则

**默认行为**：不搜索、不下载、不嵌入任何网络图片。

**触发条件**：仅当用户明确指示（如"搜一张现场图""帮我找配图"）时才执行。

**替代方案**：用户自行提供图片文件路径，引擎负责嵌入到报告中。

**图片来源链式降级**：
1. Wikimedia Commons（CC/PD 许可，API 直接返回许可证字段 — 零成本）
2. 中国政府网 / 新华网 / 人民网 等官媒新闻配图（引述性使用，标注来源）
3. Unsplash / Pexels（免费商用许可，零成本，但中文新闻素材少）
4. 以上均无结果 → 跳过，不出图，输出「建议搜索关键词」供用户手动查找

**红线**：
- 不爬取个人微博/公众号/自媒体配图（肖像权风险）
- 不使用无法验证许可证的图片
- 所有嵌入图片必须标注 `[来源：机构 | 许可证 | 原始链接]`

### 6.3 编码生图规则

- DIAGRAM JSON 生图不受 6.2 限制，按分析需求正常生成
- 不要求每节必配图——分析内容为纯逻辑推演时可跳过
- 当利益关系/权力流动/叙事扩散等需要可视化时，优先使用关系网络图
